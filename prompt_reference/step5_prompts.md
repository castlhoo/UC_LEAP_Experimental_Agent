# Step 5 Prompt Reference

Step 5 does not call GPT.
It consumes Step 4 classification output and materializes the final local corpus.

## Active Role

Step 5 is the final storage and organization stage.
It selects papers, copies PDFs/data/scripts/annotations into a stable folder layout, writes paper-level sidecars, and saves `manifest_latest.json`.

## Inputs

- `step4/output/step4_classification_latest.json`
- Step 4 per-paper download/classification paths recorded in that JSON
- `step5/config/step5_config.yaml`

## Outputs

- `step5/organized/manifest_latest.json`
- organized paper folders under:
  - `step5/organized/Both/`
  - `step5/organized/Type1/`
  - `step5/organized/Type2/`
  - `step5/organized/Neither/`

## Final Paper Folder Layout

```text
<paper_folder>/
  paper_dataset_summary.json
  reasoning.json
  reasoning.txt
  pdf/
    paper_main.pdf
    paper_supplementary.pdf
    paper_peer_review.pdf
  type1/
  type2/
  scripts/
  annotations/
```

Generated sidecars live at the paper-folder root:

- `paper_dataset_summary.json`
- `reasoning.json`
- `reasoning.txt`

The `annotations/` folder is reserved for original README, metadata, manifest, or description files from the dataset, not generated pipeline sidecars.

## Selection Modes

Step 5 supports these selection modes in config:

- `both_types`: organize only papers containing both Type 1 and Type 2.
- `has_any`: organize papers containing either Type 1 or Type 2, balanced across Both / Type1 / Type2 when `max_papers` is set.
- any other mode: organize from all Step 4 papers subject to `max_papers`.

## Rule-Based Routing

Step 5 routes files using Step 4 classification plus deterministic fallback rules:

- GPT/Step 4 `type1` -> `type1/`
- GPT/Step 4 `type2` -> `type2/`
- GPT/Step 4 `script` or script extension -> `scripts/`
- GPT/Step 4 `documentation`, README, manifest, description files -> `annotations/`
- PDFs -> `pdf/`
- Step 4 generated `paper_dataset_summary.json` -> paper-folder root
- Step 4 assessment/classification batch internals -> skipped
- unclassified archives and unclassified data-like leftovers -> skipped rather than silently misclassified

## Reasoning Sidecars

### `paper_dataset_summary.json`

Copied from Step 4 and placed at the paper-folder root.
It contains the combined paper, URL, dataset, inspection, and classification dossier.

### `reasoning.json`

Generated by Step 5 at the paper-folder root.
It includes:

- paper metadata,
- why the paper was selected,
- source URLs,
- scientific summary and dataset meaning,
- final Type 1 / Type 2 decision,
- file lists and per-file reasoning,
- file counts.

It does not include `priority_score`.

### `reasoning.txt`

Generated by Step 5 at the paper-folder root.
It is a human-readable companion to `reasoning.json` and is meant for quick manual inspection.

## Important Non-Prompt Note

Step 5 is deliberately rule-based.
If classifications look wrong, fix Step 4 classification logic or move files manually in the final organized folder; Step 5 should not invent new GPT judgments.
