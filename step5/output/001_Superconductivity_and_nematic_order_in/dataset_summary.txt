c. Notes for Analysis
- Structure patterns
  - Several Excel files store multiple traces side-by-side in alternating columns, with repeated axis/value headers
  - Some sheets contain only numeric grids with implicit axes, suggesting image-like data rather than simple line plots
  - The `.sxm` files are raw instrument outputs and likely require specialized parsing
- Potential challenges / ambiguities
  - Exact physical meaning of the grid axes in `Fig2d/e/g/h/i/k`, `Fig3c/h` is not explicit from metadata alone
  - The `.sxm` channel names, scan sizes, bias voltages, and temperatures are not provided
  - Some spreadsheets mix labels and data in the same rows, so careful parsing is needed
- Suggestions for analysis
  - For XRD: identify peak positions, compare with expected lattice structure, and assess phase purity
  - For magnetization/transport: extract superconducting onset, transition width, and field dependence
  - For grid/map data: treat as 2D arrays for image analysis, symmetry analysis, Fourier transforms, or line cuts
  - For `.sxm`: inspect metadata headers first, then analyze topography, dI/dV maps, and QPI anisotropy
  - Given the paper context, prioritize analyses related to superconducting gap structure and nematic symmetry breaking
