# collected_papers — Condensed Matter Dataset Collection

20 condensed-matter physics papers with their open datasets, organised
into Type 1 / Type 2 / Both buckets by classification.

The actual paper folders (PDFs + dataset files) live in a separate
source paper directory outside this repo. The `type1/`, `type2/`, and
`both/` subfolders here are **symlinks** that point at the per-paper
folders inside that source dir, so the data is stored exactly once.
Edit a file in the source dir and the change is visible from the
corresponding bucket here immediately.

## Contents

```
collected_papers/
├── README.md
├── collection_summary.json     authoritative metadata for all 20 papers
├── type1/                      labelled / processed / replotting-ready
│   ├── P01_bismuth_quantum_oscillations -> <paper-source>/P01_...
│   └── ...
├── type2/                      raw or binary instrument data
│   ├── P02_kondo_insulator_SmB6 -> <paper-source>/P02_...
│   └── ...
├── both/                       deposits with both kinds of content
│   ├── P04_QAH_insulator_transport -> <paper-source>/P04_...
│   └── ...
├── type1.zip                   archive snapshot of type1/
├── type2.zip                   archive snapshot of type2/
├── both.zip                    archive snapshot of both/
└── scripts/                    older copy of the organize scripts
                                (canonical copies live in ../pipeline/)
```

The pipeline that built and maintains this folder is at
[`../pipeline/`](../pipeline) (sibling of this folder under
`UC_LEAP_Experimental_Agent/`).

## What classification means

| bucket  | what's inside                                                       |
|---------|---------------------------------------------------------------------|
| `type1` | labelled CSV / TSV / TXT / xlsx files with physical-quantity column headers, figure-organised, ready to replot. Microscopy images (.tif, .tiff, .gwy), DFT outputs, and Monte Carlo results are also Type 1 unless the paper or README explicitly calls them raw. |
| `type2` | raw or binary instrument files (.h5, .npy, .pkl, .3ds, .sxm, .mat, .spe, .ibw, .tof, …) and unlabelled `.dat` measurement files. Loading these requires analysis scripts. |
| `both`  | deposits that bundle both — usually a `raw data/` subfolder plus a sibling `data in plots/` or processed CSV. |

`collection_summary.json` is the authoritative metadata file. For each
paper it records: title, authors, year, journal, DOI, arXiv ID, dataset
URL, file listing, classification, classification reason, extracted
data-availability statements from the PDF, and any v2 revision notes.

## Rebuilding this folder from scratch

Everything here is regenerated by the scripts in
[`../pipeline/`](../pipeline). The relevant ones for this folder are in
phase 05:

```bash
# from dataset_collection/UC_LEAP_Experimental_Agent/
python pipeline/05_organize/organize_by_classification.py
python pipeline/05_organize/extract_dataset_zips.py
python pipeline/05_organize/verify_layout.py
```

`organize_by_classification.py` reads `collection_summary.json` and
recreates the symlinks. `verify_layout.py` confirms the on-disk layout
matches the JSON. Both are idempotent.

See [`../pipeline/`](../pipeline) for the full discover →
download → inspect → classify → refine → organize workflow.

## v2 revision history

`collection_summary.json` carries a top-level `revision_notes` array
documenting the v2 changes against the original collection:

* P01 author list corrected against the arXiv preprint (Chen and Wu are
  the co-first authors; Tulu is third).
* All 20 PDFs re-scanned end-to-end (body + references) for
  Zenodo/figshare citations, not just Data Availability sections — new
  evidence added for P08, P14, P16; negative scans documented for P01
  and P10.
* Microscopy images (`.tif`, `.tiff`, `.gwy`) assumed processed (Type 1)
  unless the paper, README, or Zenodo description explicitly says
  otherwise. Affects P01, P16, P18, P20.
* DFT outputs and Monte Carlo simulation results deposited for figure
  reproduction treated as Type 1. Affects P16.
* Preprint PDFs are used throughout because the published versions are
  copyrighted; published DOIs are retained in metadata.
* P09 (Graphene-CrSBr spintronics) `data.zip` was extracted in place to
  `dataset/type2/extracted/` (the zip was intact but had not been
  unpacked, which made the dataset folder appear empty). P09 was later
  reclassified to `Type1+Type2` because the deposit bundles both raw
  `.dat` instrument files and per-figure plotting scripts/outputs.
